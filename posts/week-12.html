Week 12 - Progress Update

What We Achieved This Week

This week we made significant progress in enhancing functionality, improving performance, and addressing design challenges. Here's a summary of what we accomplished:

  • Event Section Implementation: We added functionality for the events tab in the menu, allowing users to view upcoming space events like eclipses and lunar events.
  • Collision Prediction: We began implementing the collision prediction feature. While it is not yet complete, the groundwork has been laid for this crucial functionality.
  • Database Schema Update: The database schema was updated to accommodate new features and improve data handling.
  • UI Bug Fixes: A number of bugs in the user interface were identified and resolved, resulting in a smoother user experience.
  • Events Tab Design: Work is ongoing on a new design for the events tab, ensuring it is both intuitive and visually appealing.
  • SPG4 Visualization: A major change was implemented by moving SPG4 calculations to the client side in the visualizer. This change reduces server load and speeds up calculations, providing a faster and more efficient user experience.
  • Real-Time Satellite Movement: Satellites now move in real time based on their trajectories, offering a dynamic and engaging visualization experience.
  • API Change for TLE Data: We transitioned from using ny2o to Space-Track.org for fetching TLE data. This change allows for unlimited API calls, improving data availability and system reliability.
